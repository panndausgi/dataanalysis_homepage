# データ分析アプリ モックアップ＆MVP 開発まとめ

本プロジェクトは、個人店舗や飲食店の店主が直感的に営業データや廃棄ロスデータを記録し、カレンダー形式で可視化できるモバイルファーストのデータ分析アプリケーション（MVP）です。

---

## 📂 プロジェクト構成

```text
Data_analysis_tool/
├── index.html     # フロントエンド（HTML / CSS / Vanilla JS）
├── main.py        # バックエンドAPI（Python / FastAPI）
├── data.db        # SQLite データベース（サーバー起動時に自動生成）
└── README.md      # 本まとめドキュメント
```

---

## 🛠️ 技術スタック

- **フロントエンド**: HTML5, Tailwind CSS v4 (CDN), Vanilla JavaScript
- **バックエンド**: Python 3.11+, FastAPI, Uvicorn (ASGIサーバー)
- **データベース**: SQLite3 (標準ライブラリ)
- **フォント**: Outfit (英数), Noto Sans JP (日本語)

## ✨ 実装された機能と画面

### 【画面1】トップページ (Splash)
- 中央にデータ分析を象徴するポップな3D風のSVGグラフィックを配置。
- 「はじめる」ボタンと「履歴カレンダーを見る」ボタンを配置。

### 【画面2】用途選択ページ (Goal Selection)
- 「お店を出したい」（新規店舗ルート）と「売り上げを改善（ロス削減）」（既存店舗ルート）を縦並びで配置。
- 各ボタンにはイラストに合わせたカラフルなアイコンと分かりやすい補足説明テキストを付与。

### 【画面3】日々のデータ入力画面 (Daily Input)
- **日付の自動設定**: 開いた時点で本日の日付をデフォルト値としてセット。
- **天気の簡単選択**: ☀️ ☁️ ☔ ❄️ のアイコン付きセグメントボタンをタップするだけで入力可能。
- **客数・廃棄数カウンター**: 大きな「＋」「ー」ボタンを配置し、タップのみで数値を増減可能。
- **売上のクイック入力**: 「+1万円」「+5万円」のショートカットと、自動テンキー立ち上げ（`inputmode="numeric"`）で入力を省力化。
- **保存トースト演出**: 「保存」押下時に「データを保存しました！」とフェードインするトーストを表示し、自動でカレンダー画面に遷移。

### 【画面4】履歴カレンダー画面 (Calendar Dashboard)
- 前月・翌月の月切り替え機能と、表示する月ごとのデータをAPIから動的取得（CORS対応）。
- データが入力されている日付セルには、「✅」アイコンと売上額（例: `5.2万`）をバッジとして表示。
- データが存在するセルをタップすると、その日の「客数」「天気」「売上」「仕込みパン廃棄数」「日替わり弁当廃棄数」を一覧できる **美しい詳細モーダルポップアップ** を表示。

## 🚀 起動・動作確認手順

### 1. 依存ライブラリのインストール
初めて起動する場合は、ターミナルで以下のコマンドを実行してください。
```bash
pip install fastapi uvicorn
```

### 2. APIサーバー（バックエンド）の起動
サーバープログラムがあるディレクトリで、以下のコマンドを実行します。
```bash
uvicorn main:app --reload
```
- サーバーが `http://127.0.0.1:8000` で起動します。
- インタラクティブなAPI仕様書は `http://127.0.0.1:8000/docs` から確認可能です。

### 3. アプリケーションの閲覧
- バックエンドが起動した状態で、`index.html` をブラウザ（Chromeなど）で開くことで、データ保存・カレンダー表示の連携動作を確認できます。

## 👥 AI分業チーム体制（今後の拡張用）

本プロジェクトは以下の4つのサブエージェント役割を設計し、それぞれ分業して進められる体制を整えています。

1. **`ui_ux_designer`** (UI/UXデザイナー / フロントエンド)
   - モバイルファーストUIの実装、チャート描画、入力フォームの最適化、状態遷移のアニメーション作成など。
2. **`backend_engineer`** (DB・バックエンド)
   - データベース設計、APIエンドポイントの構築、サーバー構成、CORSなどのセキュリティ設定。
3. **`data_scientist`** (データサイエンティスト / 予測ロジック)
   - 将来的な「廃棄予測」「黒字ラインの計算」「需要予測」のためのアルゴリズム・統計モデルの設計。
4. **`qa_checker`** (品質保証 / テスト)
   - コードのバグ検証、結合テスト、UIとバックエンド間の通信の矛盾チェックと修正。
